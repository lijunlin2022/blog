# ORM

程序员在使用数据库时，都尝试过对操作数据库的代码进行封装和优化，比如 Java 的 JDBC  就有多种封装。

一般可以大致分为两种思想：

- 着重对原始 API 层进行抽取和封装，并增强功能，典型的比 Apache 的 DbUtils，在 JDBC 上进行了 API 层的抽取和封装
- 第二种，使用面向对象的方式，让程序员以操作对象的方式操作数据库，无需编写 sql 语句，典型代表是 ORM（Object Relational Mapping），把对 sql 的操作转换为对对象的操作，从而让程序员使用起来更加方便和已于接受，这种转换是通过对象和表之间的元数据映射实现的，这是实现 ORM 的关键，如下图所示
